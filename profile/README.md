## 搬瓦工 IP 被封，到底是怎么回事？

先说清楚概念。IP 被封，本质上是这个 IP 地址在国内网络环境中被列入屏蔽名单，来自国内的访问请求被直接阻断。这种封锁发生在网络层，不是搬瓦工官方主动封禁你的账户，所以你的 KiwiVM 面板里服务器状态一切正常，但就是从国内连不上。

原因大致有这几类：

**1. 自身使用方式触发**

流量特征过于明显，协议特征容易被识别，或者短时间内出现异常流量、高频连接行为，都可能触发风控机制。这是最常见的原因。

**2. IP 历史遗留问题**

你拿到手的这个 IP，可能在之前某个用户手里就已经"声名狼藉"了，被纳入了黑名单，只是还没走完下架流程就又分配出来了。和你本人的使用方式完全无关，就是运气不好。

**3. IP 段连坐**

同一个 IP 段里，某些用户做了触发屏蔽的操作，整个 IP 段被整体拉黑。即使你自己什么都没做，也会被一起封掉。这个没有任何预防手段，纯粹是"池鱼之殃"。

**4. 特殊时期集中清理**

某些时间节点，GFW 会进行集中性的大规模封锁，大量 IP 短时间内同时失效。很多老用户都经历过"某天起来突然全断了"的情况。

## 怎么确认是 IP 被封，而不是其他问题？

在折腾换 IP 之前，先确认一下是不是真的被封了。有两个方式：

**方式一：第三方工具检测**

使用 IPCheck.ing 或类似工具，输入你的服务器 IP 和 SSH 端口，看一下国内和国外的连通性差异。如果国外 TCP 正常、国内 TCP 不通，基本可以确认是 IP 被封；如果连 ICMP 都挂了，那是全面封锁。

**方式二：搬瓦工官方检测工具**

登录 KiwiVM 控制面板，打开检测链接（路径：主面板 → Blacklist check），点击 Test Main IP。显示 `IP NOT BLOCKED` 说明 IP 在官方检测范围内可用；如果显示被封，就得考虑下面的解决方案了。
